# Organizational Structure

Every resource inside ENQI belongs to an Organization.

Departments and Workspaces are independent structures within the same Organization.

Departments represent the formal organizational structure of the customer.

Workspaces represent business, project or knowledge contexts where authorized users collaborate.

```text
Organization
│
├── Departments
│   ├── Finance
│   ├── Legal
│   ├── Sales
│   └── Human Resources
│
├── Users
│
├── Roles
│
├── Permissions
│
└── Workspaces
    ├── Contracts
    ├── Annual Audit
    ├── Strategic Planning
    └── Acquisition Project
```

A Workspace does not belong to a Department.

A Department does not own a Workspace.

Users from different Departments may collaborate inside the same Workspace when explicitly authorized.

Access to every Workspace and resource is determined by:

- Organization membership
- Workspace membership
- Assigned Roles
- Permissions
- Department context, when applicable
- Resource classification
- Explicit access grants
- Governance policies

This model preserves organizational structure without limiting cross-department collaboration.

# Architectural Principles

The architecture of ENQI follows the following principles:

- Domain-Driven Design (DDD)
- Clean Architecture
- Modular Monolith (initially)
- API First
- Security by Design
- AI by Context
- Multi-Tenant by Design
- Workspace Isolation
- Event-Driven Evolution
- Cloud-Native Ready
